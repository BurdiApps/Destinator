# Destinator — Admin Dashboard (Cloud Databases Module)

A Flask + Google Firestore admin panel that displays and manages GPS trip data generated by my C++ Destinator simulator. Built for CSE 310: Applied Programming — Cloud Databases Module.

## Cloud Database

**Google Firestore** (Firebase) — a NoSQL key/value cloud database.

## Description

Destinator is a modular GPS trip-logging system. The C++ simulator generates realistic trip segments with waypoint interpolation, haversine-based mileage, and odometer comparison. This admin dashboard connects to Firestore to store, view, edit, and delete trip data as if an administrator is monitoring drivers' GPS devices in real time.

The admin UI is a dark-themed, purple-accented dashboard with a persistent sidebar, data tables, stat cards, and CRUD forms. I chose this layout style because it keeps everything accessible from one screen.

### Firestore Collections (Three Related Tables)

| Collection | Purpose | Key Fields |
|------------|---------|------------|
| `drivers`  | Registered drivers/vehicles | `name`, `vehicle`, `created_at` |
| `trips`    | Trip segments logged by the simulator | `driver_id` (FK → drivers), `from_location`, `to_location`, `odo_start`, `odo_end`, `gps_miles`, `criteria`, `date`, etc. |
| `users`    | Admin accounts with hashed passwords | `firstname`, `lastname`, `email`, `password` (bcrypt hash) |

## Module Requirements Fulfilled

- **Cloud database**: Google Firestore (NoSQL)
- **Insert**: Add trips manually or import from C++ simulator CSV; register new users
- **Modify**: Edit any trip or driver record
- **Delete**: Remove trips or drivers (with referential cleanup)
- **Retrieve / Query**: List all trips, filter by driver, authenticate users by email lookup
- **Additional requirement**: Three related tables (`drivers` ↔ `trips` via `driver_id`, `users` for auth)
- **User authentication**: Registration and login with bcrypt password hashing (10 salt rounds), Flask-Login session management
- **UI**: Dark/light theme toggle with localStorage persistence, purple-accented design

## Setup & Installation

1. **Clone the repository:**
   ```bash
   git clone https://github.com/yourusername/destinator.git
   cd destinator/admin
   ```

2. **Create a virtual environment and install dependencies:**
   ```bash
   python -m venv venv
   source venv/bin/activate   # or venv\Scripts\activate on Windows
   pip install -r requirements.txt
   ```

3. **Set up Firebase:**
   - Go to [Firebase Console](https://console.firebase.google.com/)
   - Create a project → Create a Firestore database
   - Go to Project Settings → Service Accounts → Generate new private key
   - Save the JSON file as `admin/serviceAccountKey.json`
   - **Do NOT commit this file to GitHub** (it's in `.gitignore`)

4. **Run the Flask server:**
   ```bash
   python app.py
   ```
   Open `http://localhost:5000` in your browser.

5. **Import C++ simulator data:**
   - Run the C++ simulator to generate `trip_data.csv`
   - Use the admin UI: go to **Import CSV** and upload the file
   - Or use the CLI: `python csv_importer.py ../trip_data.csv`
